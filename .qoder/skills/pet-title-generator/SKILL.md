# 宠物庙号谥号生成器

根据宠物的 pet-info.ts 中的描述信息，自动生成 `title` 字段下的庙号、谥号等封号信息，并保存回 pet-info.ts 文件。

## 调用方式

### 独立调用

用户直接提供宠物 ID。适用于单独生成或重新生成庙号谥号的场景。

### 被主流程调用

当从 `pet-onboarding` 主流程调用时，通过 `Skill` 工具传入 `petId` 参数，流程与独立调用相同。注意：被主流程调用时 title 字段不存在，无需询问用户是否覆盖。

## 重要规则

- **只生成/修改 `title` 字段**，绝不修改 pet-info.ts 中的其他任何字段
- 如果 `title` 字段已存在，需要询问用户是否覆盖
- 生成的内容应幽默、有趣、充满爱意，符合"赛博墓碑"项目的调性

## 输入要求

用户提供宠物 ID（即 `src/app/pets/[petId]/` 目录名），例如 `wangcai`、`xiaoxue`。

## 输出规格

生成 `PetTitle` 接口定义的 5 个字段：

| 字段 | 说明 | 示例 |
|-----|------|------|
| `templeName` | 庙号（X宗/X祖） | 馋宗、傲宗 |
| `templeNameNote` | 庙号释义 | 一生以嘴为尊... |
| `posthumousName` | 谥号（四字词+宠物名+尊号） | 忠烈贪嘴摇尾如风黏人圣善旺财大帝 |
| `posthumousNameNote` | 谥号释义（逐词解释） | 忠烈——守门十二载... |
| `fullTitle` | 全称（朝代 + 庙号 + 谥号） | 大旺朝 馋宗 忠烈贪嘴摇尾如风黏人圣善旺财大帝 |

## PetTitle 接口定义

```typescript
export interface PetTitle {
  templeName: string;        // 庙号
  templeNameNote: string;    // 庙号释义
  posthumousName: string;    // 谥号
  posthumousNameNote: string;// 谥号释义
  fullTitle: string;         // 全称
}
```

**注意：** 以上接口定义仅供快速参考。权威定义在 `src/app/models/pet.model.ts`，如有差异以 model 文件为准。

## 生成流程

### 步骤 1：读取宠物信息

使用 Read 工具读取 `src/app/pets/[petId]/pet-info.ts`，提取以下字段用于生成：
- `name`（名字）
- `species`（物种：犬/猫/兔/鸟等）
- `breed`（品种）
- `gender`（性别：男/女）
- `personality`（性格标签数组）
- `favoriteThings`（喜好数组）
- `bio`（生平描述）
- `ownerMessage`（主人寄语）
- `epitaph`（墓志铭，如果有的话）

### 步骤 2：检查现有 title 字段

如果文件中已存在 `title` 字段且有内容，**必须询问用户是否覆盖**，不可直接替换。

### 步骤 3：生成庙号谥号

**信息不足时的处理策略：**
- 如果 `personality` 少于 3 项，从 `bio` 和 `ownerMessage` 中挖掘隐含的性格特质
- 如果 `bio` 过短（少于 30 字），结合 `species` + `breed` 的典型行为特征来补充素材（如柯基 → 短腿、金毛 → 忠诚友好）
- 如果 `favoriteThings` 稀疏，从 `photos` 的 caption 或 `bio` 内容中推断喜好
- 绝不捏造无法从任何输入字段推导出的信息，宁可用品种通用特征也不要凭空编造具体事件

基于提取的宠物信息，按照以下规则创意生成：

#### 庙号（templeName）规则
- 默认格式：**单字 + "宗"**（如：馋宗、傲宗、懒宗、萌宗）
- `"祖"` 保留给特殊宠物：家中第一只宠物、超长寿（15 年以上）、或 bio 中有明确"开创/先驱"含义的宠物（如：开祖、寿祖）。不确定时默认用"宗"
- 选取宠物**最突出的一个性格特征**作为庙号核心字
- 用字要精准传神、带有戏谑感

#### 庙号释义（templeNameNote）规则
- 一句话解释庙号由来
- 要结合宠物的具体行为习惯
- 风格：**幽默夸张 + 拟人化**
- 包含一个用「」括起来的"绝技"或"奥义"名称
- 示例：`一生以嘴为尊，见食物双眼放光，修炼出「隔空闻骨」绝技`
- 示例：`一生高贵冷艳，仅在罐头开启时短暂降尊屈贵，修炼出「已读不回」奥义`

#### 谥号（posthumousName）规则
- 格式：**多个二字/四字描述词 + 宠物名 + 尊号**
- 描述词从 personality 和 favoriteThings 中提炼，每个词概括宠物的一个特质或趣事
- 尊号根据 gender 选择：
  - 男：大帝 / 圣皇
  - 女：女皇 / 圣后
- 整体谥号长度建议 15-25 字，最长不超过 30 字。特质过多时优先选择最鲜明的 3-4 个，不必面面俱到
- 示例：`忠烈贪嘴摇尾如风黏人圣善旺财大帝`
- 示例：`慧静高冷偶尔撒娇窗台圣洁小雪女皇`

#### 谥号释义（posthumousNameNote）规则
- 逐词（每 2-4 字为一组）解释谥号含义
- 格式：`词1——解释1；词2——解释2；...`
- 每个解释要结合宠物真实行为，**具体、有画面感**
- 示例：`忠烈——守门十二载，仅对快递员狂吠不止；贪嘴——曾一口吞下整根火腿肠，连包装都没放过`

#### 全称（fullTitle）规则
- 格式：`大[X]朝 [庙号] [谥号]`
- `[X]` 为朝代代号，规则：
  - 犬类：用宠物名字中的一个字，或与犬相关的趣味字（如旺、汪）
  - 猫类：用"喵"或宠物名字中的一个字
  - 其他物种：用物种相关的趣味字
- 示例：`大旺朝 馋宗 忠烈贪嘴摇尾如风黏人圣善旺财大帝`
- 示例：`大喵朝 傲宗 慧静高冷偶尔撒娇窗台圣洁小雪女皇`

### 步骤 4：写入 pet-info.ts

使用 Edit 工具**仅修改 title 字段**：

- 如果文件中没有 `title` 字段：在合适位置（`theme` 字段之后）添加 `title` 字段
- 如果文件中已有 `title` 字段：替换 `title` 对象的内容

**写入格式示例：**
```typescript
  title: {
    templeName: '馋宗',
    templeNameNote: '一生以嘴为尊，见食物双眼放光，修炼出「隔空闻骨」绝技',
    posthumousName: '忠烈贪嘴摇尾如风黏人圣善旺财大帝',
    posthumousNameNote: '忠烈——守门十二载，仅对快递员狂吠不止；贪嘴——曾一口吞下整根火腿肠，连包装都没放过；摇尾如风——尾巴摆动频率可发电；黏人——超过三分钟不摸就开始哼唧',
    fullTitle: '大旺朝 馋宗 忠烈贪嘴摇尾如风黏人圣善旺财大帝'
  },
```

### 步骤 5：验证结果

生成完成后：
1. 使用 Read 工具重新读取文件，确认 title 字段已正确写入
2. 确认其他字段未被修改
3. 向用户展示生成的庙号谥号信息

## 风格要求

生成的内容必须具备：
- **幽默感**：用戏谑的古代帝王封号语气描述宠物日常
- **拟人化**：将宠物行为用古代官方语言包装
- **温情底色**：幽默之中饱含对宠物的深厚感情
- **具体化**：每个描述都要有具体的行为画面，避免空洞
- **个性化**：充分利用宠物的独特性格和喜好，不要泛泛而谈

## 风格禁区（避免事项）

- 避免使用任何侮辱性、贬义的真实历史称号（如"暴君"、"亡国之君"相关词）
- 避免过于笼统的描述词（如"好"、"乖"、"可爱"本身），需将其具体化为有画面感的行为描述
- 庙号避免使用真实历史上负面评价的帝王庙号（如"炀"、"厉"、"哀"）
- 谥号中的每个描述词必须能从输入信息（personality/favoriteThings/bio/ownerMessage）中推导，禁止使用无法从输入推导的特质
- 全称中的朝代名不要使用真实历史朝代名（如"大唐"、"大宋"、"大明"），应使用与宠物相关的趣味字
